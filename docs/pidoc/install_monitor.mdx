# 随时掌握你的树莓派状态

`PiDoc` 是由本站提供的树莓派CPU负载、网络、磁盘、温度等监测的一站式信息工具，还可以通过本站公网堡垒机中转 ssh 的方式随时访问树莓派。适用于持续运行树莓派的用户或是使用树莓派的开发者。如您有一台或多台树莓派，通过监测工具可很好的随时掌握其运行状态。

交流QQ群：498908352 (反馈问题、提供建议等)

* 实时监控：随时查看设备状态
* 网页SSH：随时随地远程访问
* 堡垒SSH：高级功能应用

## 网页SSH

为了解决树莓派在内网环境中无法控制的问题，`PiDoc`工具开发了 `网页SSH` 功能。在页面左侧导航栏中点击 `SSH连接` 会打开一个网页SSH页面新窗口。

请根据您设备的实际情况选择填写 **用户名**、**端口**，选择 **密码认证** 或 **私钥认证**。

<img src="/img/pidoc/web_ssh.png" width="80%" alt="网页SSH" />

## 堡垒SSH

如果您想使用 SSH 的 scp 传输文件，或是想通过 VSCode等等 IDE远程访问树莓派，可以使用 `堡垒SSH` 功能。

堡垒SSH 本质上是一个 SSH 代理，它会将您的请求转发到树莓派上。对于该功能的使用需要您有一定的SSH配置知识，本处将不赘述。

SSH config文件示例：
```
Host rpi-5
    User pi
    Port 22
    ProxyJump remote.pidoc.cn

Host remote.pidoc.cn
    HostName remote.pidoc.cn
    Port 64422
```

信息解读：
```
Host rpi-5                      #堡垒使用主机别名进行连接（设备管理中的第一列，别名可以更改），这里我们使用 `rpi-5`。
    User pi                     #树莓派SSH用户名
    Port 22                     #树莓派SSH端口
    ProxyJump remote.pidoc.cn   #指定代理跳转的主机配置项，这里我们使用 `remote.pidoc.cn`。

Host remote.pidoc.cn            #代理跳转主机下的详细配置
    HostName remote.pidoc.cn    #代理主机域名，这里我们使用 `remote.pidoc.cn`。
    Port 64422                  #代理主机端口，这里我们使用 `64422`。
```

在页面左侧导航栏中点击 `堡垒密钥` 会打开一个堡垒密钥的列表页面。你可以在此页面管理堡垒密钥。
