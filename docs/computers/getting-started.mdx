# 入门

## 开始使用你的树莓派

要开始使用树莓派，您需要以下各项：

- 一个[电源](#电源)
- 启动介质（例如[具有充足存储空间和速度的microSD卡](#推荐的sd卡)）

您可以将树莓派设置为带有桌面的交互式计算机，或者设置为只能通过网络访问的无需显示器的计算机。如将树莓派设置为无需显示器，您不需要任何额外的外围设备；您可以在安装操作系统时预先配置主机名、用户帐户、网络连接和ssh 。如果您想直接使用树莓派，则需要以下附加配件：

- 一台显示器
- 一根连接到显示器的视频线缆
- 一个键盘
- 一个鼠标

### 电源
下表显示了为各种树莓派型号供电所需的USB-PD电源模式。您可以使用任何提供正确电源模式的高质量电源。

| 型号 | 推荐电源（电压/电流） | 树莓派电源 | 
| -- | -- | -- |
| 树莓派500+ | 5V/5A | [27W USB-C 电源](https://www.raspberrypi.com/products/27w-power-supply/) |		
| 树莓派5 和 树莓派500 | 5V/5A、5V/3A(外设电流限制 600mA) | [27W USB-C 电源](https://www.raspberrypi.com/products/27w-power-supply/) |		
| 树莓派4B 和 树莓派400 | 5V/3A | [15W USB-C 电源](https://www.raspberrypi.com/products/type-c-power-supply/) |
| 树莓派3（所有型号） | 5V/2.5A | [12.5W Micro USB 电源](https://www.raspberrypi.com/products/micro-usb-power-supply/) |
| 树莓派 2B | 5V/2.5A | [12.5W Micro USB 电源](https://www.raspberrypi.com/products/micro-usb-power-supply/) |
| 树莓派1（所有型号） | 5V/2.5A | [12.5W Micro USB 电源](https://www.raspberrypi.com/products/micro-usb-power-supply/) |
| 树莓派Zero（所有型号） | 5V/2.5A | [12.5W Micro USB 电源](https://www.raspberrypi.com/products/micro-usb-power-supply/) |

<img src="/img/computers/getting-started/peripherals/cable-power.png" alt="将电源插入树莓派" width="80%" />

将电源插入标有"POWER IN"、"PWR IN"或"PWR"的端口。某些树莓派型号（例如Zero系列）具有与电源端口形状相同的USB输出端口。请确保在树莓派上使用正确的端口！

### 启动介质
树莓派模块缺乏板载存储，因此您必须提供它。您可以从树莓派支持的任何介质启动操作系统映像：通常是microSD卡，还可以是USB存贮设备、网络存储以及通过PCIe HAT连接的存储。但是，只有最新的树莓派型号支持所有这些媒体类型。

自 树莓派1 Model A+ 以来的所有树莓派消费型号均配有microSD插槽。当microSD插槽包含卡时，您的树莓派会自动从该插槽启动。

<img src="/img/computers/getting-started/peripherals/sd-card.png" alt="将SD卡插入树莓派" width="80%" />

#### 推荐的SD卡
我们建议安装 Raspberry Pi OS 时使用至少 32GB 存储容量的 SD 卡。对于 Raspberry Pi OS Lite，我们建议至少使用 16GB。您可以使用任何容量小于 2TB 的 SD 卡。由于 [MBR](https://en.wikipedia.org/wiki/Master_boot_record) 的限制，目前不支持超过 2TB 的容量。与其他启动媒体一样，SD 卡的读写速度更快，性能更好。

如果不确定购买哪种 SD 卡，请参考 [Raspberry Pi 的官方 SD 卡](../accessories/sd-cards.mdx)。

由于硬件的限制，以下设备只能从256GB或更少的启动分区启动：

- 树莓派Zero
- 树莓派1
- 早期树莓派2（使用BCM2836 SoC的型号）

其他操作系统有不同的要求。检查操作系统的文档以了解容量要求。

### 键盘
您可以使用树莓派上的任何USB端口连接[有线键盘](https://www.raspberrypi.com/products/raspberry-pi-keyboard-and-hub/)或USB蓝牙接收器。

<img src="/img/computers/getting-started/peripherals/cable-key.png" alt="将键盘插入树莓派" width="80%" />

### 鼠标
您可以使用树莓派上的任何USB端口连接[有线鼠标](https://www.raspberrypi.com/products/raspberry-pi-mouse/)或USB蓝牙接收器。

<img src="/img/computers/getting-started/peripherals/cable-mouse.png" alt="将鼠标插入树莓派" width="80%" />

### 显示

树莓派各型号具有以下显示连接：

|型号 | 显示输出 |
| -- | -- |
| 树莓派5、400、500、500+ | 2个micro HDMI |
|树莓派4B | 2个micro HDMI、音频和复合输出通过3.5mm[TRRS耳麦插孔](https://en.wikipedia.org/wiki/Phone_connector_(audio)#TRRS_standards) |
| 树莓派3（所有型号） | 1个HDMI、音频和复合输出通过3.5mm[TRRS耳麦插孔](https://en.wikipedia.org/wiki/Phone_connector_(audio)#TRRS_standards) |
| 树莓派2B | 1个HDMI、音频和复合输出通过3.5mm[TRRS耳麦插孔](https://en.wikipedia.org/wiki/Phone_connector_(audio)#TRRS_standards) |
| 树莓派1A+、1B+ | 1个HDMI、音频和复合输出通过3.5mm[TRRS耳麦插孔](https://en.wikipedia.org/wiki/Phone_connector_(audio)#TRRS_standards) |
| 树莓派1A、1B | 1个HDMI、音频通过3.5mm[TRRS耳麦插孔](https://en.wikipedia.org/wiki/Phone_connector_(audio)#TRRS_standards)，复合输出通过[phono](https://en.wikipedia.org/wiki/RCA_connector) |
| 树莓派Zero（所有型号） | 1个mini HDMI，通过焊盘输出复合视频信号 |

:::note[注意]
没有任何 Raspberry Pi 型号支持通过 USB-C 传输视频（DisplayPort 备选模式）。
:::

如果您的 Raspberry Pi 有多个 HDMI 端口，请将主显示器插入标有 "HDMI0 "的端口。

大多数显示器没有micro或mini HDMI端口。但是您可以使用[micro HDMI转HDMI线缆](https://www.raspberrypi.com/products/micro-hdmi-to-standard-hdmi-a-cable/)或[mini HDMI转HDMI线缆](https://www.raspberrypi.com/products/standard-hdmi-a-male-to-mini-hdmi-c-male-cable/)将树莓派上的这些端口连接到任何HDMI显示器。对于不支持HDMI的显示器，请考虑使用转换器将显示输出从HDMI转换到显示器支持的端口。

<img src="/img/computers/getting-started/peripherals/cable-hdmi.png" alt="将mini HDMI插入树莓派" width="80%" />

### 音频
所有配备HDMI、micro HDMI或mini HDMI的树莓派型号都支持通过HDMI进行音频输出。所有树莓派型号均支持USB音频。所有配备蓝牙的树莓派型号都支持蓝牙音频。树莓派 1、2、3 和 4 的系列型号均包含3.5mm[TRRS耳麦插孔](https://en.wikipedia.org/wiki/Phone_connector_(audio)#TRRS_standards)，可能需要功放才能获得足够的输出音量。

### 网络
以下树莓派型号配备WiFi和蓝牙连接：

- 自 树莓派3B 以后的所有旗舰型号
- 所有键盘型号
- 所有的 Zero W 型号
- 所有的 Pico W 型号
- 配置无线功能的计算模块（自 CM4 起可用）

型号后缀"B"表示配备以太网端口的变体；"A"则表示未配备以太网端口。键盘型号同样包含以太网端口。若您的树莓派未配备以太网端口，仍可通过USB转以太网适配器连接有线网络。

<img src="/img/computers/getting-started/peripherals/cable-net.png" alt="将以太网线缆插入树莓派" width="80%" />

## 安装操作系统

要使用树莓派，您需要一个操作系统。默认情况下，树莓派设备检查插入SD卡插槽的SD卡上的可用操作系统。

根据您的树莓派型号，您还可以从其他存储设备启动操作系统，包括USB存贮设备、通过HAT连接的存储和网络存储。

要在树莓派的存储设备上安装操作系统，您需要：

- 可用于将系统映像写入存储设备的计算机
- 将存储设备插入该计算机

大多数树莓派用户选择microSD卡作为启动设备。

我们推荐使用[Raspberry Pi Imager](#使用imager工具安装)安装操作系统。

如果您没有其他计算机可以将映像写入启动设备，您可以[直接从互联网在树莓派上安装操作系统](#通过网络安装)。

### 使用Imager工具安装

Raspberry Pi Imager是一款可帮助您在macOS、Windows和Linux上下载和写入映像的工具。Imager包含许多适用于树莓派的流行操作系统映像。Imager还支持加载直接从[树莓派](/downloads/operating-systems/)或第三方供应商（例如[Ubuntu](https://ubuntu.com/download/raspberry-pi)）下载的映像。您可以使用Imager为树莓派预配置登录凭据和远程访问设置。

Imager支持以以`.img`格式打包的映像以及容器格式，例如`.zip`。

#### 步骤1：安装并启动 Imager

将 Raspberry Pi Imager 下载至您的计算机（Windows、Mac 或 Linux）：.

1. 通过以下任一方式安装 Imager：
  * 在 Windows 和 Mac 上，从 [软件下载](/downloads/) 下载最新版本。运行安装程序。
  * 在树莓派操作系统上，运行 `sudo apt install rpi-imager`。
  * 在其他 Linux 系统上，从 [软件下载](/downloads/) 下载最新版本的 AppImage 文件。赋予 AppImage 可执行权限。.

2. 通过以下任一方式启动树莓派刻录工具：
  * 在应用程序菜单中查找。
  * 点击桌面上的树莓派映像工具图标。
  * 以 root 权限运行 AppImage 文件。例如使用命令：`sudo ./Raspberry_Pi_Imager-2.0.0-desktop-x86_64.AppImage`

#### 步骤2：配置基础参数

告知树莓派映像工具所需的硬件与操作系统组合：

1. 在**设备**选项卡中，从列表中选择您的树莓派型号。点击**下一步**。

    <img src="/img/computers/getting-started/imager/choose-device.png" alt="树莓派Imager设备选项卡" width="80%" />

2. 在**操作系统**选项卡中，从可用系统中选择。点击**下一步**。

    Imager会在列表顶部显示推荐的树莓派操作系统版本。
若需安装计算机中的镜像文件，请选择**使用自定义**选项。

    <img src="/img/computers/getting-started/imager/choose-os.png" alt="Imager中的操作系统选项" width="80%" />

3. 将所需存储设备连接至计算机。例如，通过外置或内置SD卡读卡器插入microSD卡。

4. 在**存储**选项卡中，选择用于写入镜像的存储设备。点击**下一步**。

    <img src="/img/computers/getting-started/imager/choose-storage.png" alt="Imager中的存储选择界面" width="80%" />

:::warning
若计算机连接了多个存储设备，_请务必选择正确的设备_！通常可通过容量大小识别存储设备。若无法确定，请断开其他设备直至确认目标设备。当您尝试覆盖计算机系统盘时，Imager会发出警告。
:::

#### 步骤3：自定义操作系统

树莓派操作系统及部分第三方操作系统支持自定义设置。若您安装的是此类操作系统，可通过Imager中的后续步骤进行个性化配置。这些步骤为可选操作，选择**跳过自定义**即可省略。但我们强烈建议您在首次启动前，通过Imager的自定义设置完成树莓派的初始配置。

可预先配置：

* 设备主机名
* 时区设置
* 键盘布局
* 用户名与密码
* Wi-Fi 凭证
* 远程连接选项
* Raspberry Pi Connect 功能

若您先前已通过 Imager 完成自定义步骤，后续操作中将自动填入您选择的值。

若未通过这些操作系统自定义设置配置树莓派，系统将在首次启动后再次要求您输入相同信息。对于树莓派操作系统，首次启动配置将在[配置向导](#首次启动配置)中进行。

1. 在 **自定义\>主机名** 选项卡中，为树莓派输入仅含字母、数字和连字符的主机名，然后选择**下一步**。

    树莓派将通过[mDNS](https://en.wikipedia.org/wiki/Multicast_DNS)协议向网络广播此主机名。当树莓派连接网络后，其他设备可通过 `<主机名>.local` 或 `<主机名>.lan` 与您的计算机通信。

    <img src="/img/computers/getting-started/imager/os-customisation-hostname.png" alt="Imager OS 自定义主机名选项卡" width="80%" />


2. 在 **自定义\>本地化** 选项卡中，选择您的首都城市。Imager将自动填充该城市的时区和键盘布局，您可修改这些设置。选择**下一步**。

    <img src="/img/computers/getting-started/imager/os-customisation-locale.png" alt="Imager OS定制化本地化选项卡" width="80%" />

3. 在 **自定义>用户** 选项卡中，为树莓派管理员用户输入用户名和密码。选择**下一步**。

    <img src="/img/computers/getting-started/imager/os-customisation-user.png" alt="Imager OS自定义用户选项卡" width="80%" />

4. 在 **自定义>Wi-Fi** 选项卡中，首次使用时Imager会预填当前连接Wi-Fi网络的SSID（名称）和密码。

    a. 选择连接 **安全网络** 或 **开放网络**。

    b. 若无预填信息或需修改，请输入无线网络的SSID（名称）。对于安全网络，需输入并确认网络密码。

    c. 若您的网络未公开广播SSID，请启用 **隐藏SSID** 设置。

    d. 选择 **下一步**。

    <img src="/img/computers/getting-started/imager/os-customisation-wifi-secure.png" alt="Imager OS自定义设置-Wi-Fi选项卡，安全网络界面" width="30%" />
    <img src="/img/computers/getting-started/imager/os-customisation-wifi-open.png" alt="Imager OS自定义设置-Wi-Fi选项卡，开放网络" width="30%" />

5. 在 **自定义\>远程访问** 选项卡中配置您的 SSH 设置。

    a. 若需通过网络远程使用树莓派，请将 *启用 SSH* 开关切换至激活状态。

    若计划运行无显示器的树莓派，请启用此选项或为设备配置树莓派连接工具。

    b. 启用 SSH 后，请选择认证方式：

      * 选择**使用密码认证**，通过**自定义\>用户**选项卡中设置的用户名和密码进行网络登录。

      * 选择**使用公钥认证**，配置基于SSH密钥的认证机制。Imager将显示公钥输入框。若SSH配置中已存在RSA公钥，Imager将自动采用该公钥。若未配置，可将公钥复制到Imager或浏览本地计算机查找公钥文件。

        <img src="/img/computers/getting-started/imager/os-customisation-ssh.png" alt="Imager操作系统定制远程访问选项卡" width="80%" />

6. 部分操作系统现已显示 **自定义\>Raspberry Pi Connect** 选项卡。要将设备关联至您的 [树莓派Connect账户](../services/connect.mdx)，请完成以下步骤：

    a. 将 **启用树莓派Connect** 开关切换至激活状态。

      <img src="/img/computers/getting-started/imager/os-customisation-connect.png" alt="Imager 操作系统定制 树莓派Connect选项卡" width="60%" />

    b. 选择 **打开树莓派Connect**。树莓派Connect网站将在默认浏览器中打开。

      <img src="/img/computers/getting-started/imager/os-customisation-site.png" alt="树莓派Connect页面" width="60%" />

    c. 使用您的树莓派ID账户登录，若尚未注册请创建账户。

      <img src="/img/computers/getting-started/imager/connect-auth-key.png" alt="树莓派连接新认证密钥页面" width="60%" />

    d. 在 **新建认证密钥** 页面创建您的认证密钥。

      该密钥为一次性临时令牌。树莓派连接网站会显示令牌自生成起有效期。使用令牌前，请确保在过期时间前启动树莓派并连接网络。

      * 若您的树莓派ID账户未加入任何组织，请选择 **创建认证密钥并启动树莓派Imager**。

      * 若您已加入一个或多个组织，请先选择关联该密钥的组织或账户，再选择 **创建认证密钥并启动树莓派Imager**。

    e. 浏览器可能询问是否允许该网站打开 Raspberry Pi Imager。请确认您希望启动 Imager。

      Raspberry Pi Imager 将自动跳转至 **Raspberry Pi Connect** 标签页。该标签页会显示一条确认信息，表明 Imager 已从浏览器接收认证令牌，并提供包含令牌的输入框。

      若令牌未自动复制，可手动将其粘贴至 Imager。打开 Raspberry Pi Connect 网页的 **遇到问题？** 部分，复制令牌并粘贴到 Imager 的令牌字段中。

      <img src="/img/computers/getting-started/imager/os-customisation-connect-token.png" alt="Raspberry Pi Connect 令牌接收提示信息" width="60%" />

    f. 选择 **下一步**。

#### 步骤4：设置Imager选项

您可选择配置 Imager 在写入过程中的行为。在写入映像前，随时可通过选择 **应用程序选项** 进行设置。

在 **应用程序选项** 对话框中，可配置以下选项：

* Imager 在写入完成时是否播放提示音
* Imager 在写入完成时是否弹出存储介质
* 是否向我们发送有关设备和操作系统的匿名遥测数据。详情参见 [匿名指标（遥测）](https://github.com/raspberrypi/rpi-imager?tab=readme-ov-file#anonymous-metrics-telemetry)。
* 写入过程中是否显示警告信息。

（高级用户）**应用选项**对话框还允许您设置自定义存储库，用于检索操作系统和设备列表。更多信息请参阅 [Imager 存储库 URL](./raspberry-pi.mdx#imager_repo_url)。

<img src="/img/computers/getting-started/imager/app-options.png" alt="应用选项对话框" width="80%" />

#### 第5步：将映像写入存储设备

完成Imager各选项卡设置后，系统将显示摘要页面供您核对选项。

1. 若确认无误，请选择**写入**。

    <img src="/img/computers/getting-started/imager/summary.png" alt="写入前在Imager中确认选项" width="80%" />

2. Imager将弹出警告提示即将覆盖存储介质上的所有数据。继续操作请选择**我已知晓，清除并写入**。

    若出现管理员权限提示，要求授予存储介质读写权限，请允许Imager继续操作。

    <img src="/img/computers/getting-started/imager/are-you-sure.png" alt="在Imager中确认存储设备重写映像" width="80%" />

    Imager开始将映像写入存储介质。此过程可能需要数分钟，请稍作休息。

    <img src="/img/computers/getting-started/imager/writing.png" alt="Imager向设备写入镜像" width="80%" />
    *Imager正在向存储设备写入镜像。*


    写入完成后，Imager将验证镜像是否正确写入。建议您完成验证步骤。若需取消验证，请选择 **跳过验证**。

    <img src="/img/computers/getting-started/imager/verify.png" alt="Imager在设备上验证映像" width="80%" />
    *写入完成后，Imager将执行验证。*


    映像写入并验证完成后，Imager将显示 **完成** 选项卡。选择 **结束** 即可退出程序。

    <img src="/img/computers/getting-started/imager/finished.png" alt="Imager向存储设备写入镜像完成后的界面" width="80%" />
    *镜像操作已完成。*

    接下来请转至 [配置树莓派](#设置你的树莓派) 章节，启动您的树莓派设备。

### 通过网络安装
网络安装使树莓派能够使用通过网络下载的树莓派Imager版本在存储设备上安装操作系统。通过网络安装，您可以在树莓派上安装操作系统，无需单独的SD卡读卡器，也不需要除树莓派之外的任何计算机。您可以在任何兼容的存储设备上运行网络安装，包括SD卡和USB存贮设备。

自 Raspberry Pi 4B 和键盘型号起，网络安装只能在旗舰型号上运行。如果你的 Raspberry Pi 运行旧版引导加载器，你可能需要 [更新引导加载器](./raspberry-pi.mdx#树莓派启动eeprom)才能使用网络安装。

网络安装需要以下内容：
- 支持网络安装固件的树莓派型号
- 一台显示器
- 一个键盘
- 有线互联网连接

要启动网络安装，请在满足以下配置的情况下按住**SHIFT**键的同时打开树莓派电源：
- 无可启动存储设备
- 连接了键盘
- 连接了兼容存储设备，例如SD卡或USB存贮设备

<img src="/img/computers/getting-started/network-install-1.png" alt="网络安装屏幕" width="80%" />

如果您尚未将树莓派连接到互联网，请使用以太网线将其连接。

<img src="/img/computers/getting-started/network-install-2.png" alt="开始网络安装" width="80%" />

连接到互联网后，您的树莓派将下载树莓派安装程序。如果下载失败，您可以重复该过程重试。

<img src="/img/computers/getting-started/network-install-3.png" alt="使用网络安装下载Imager" width="80%" />

下载完树莓派Installer后，您的树莓派将自动启动树莓派Imager。有关运行树莓派Imager的更多信息，请参阅[安装操作系统](#安装操作系统)。

<img src="/img/computers/getting-started/network-install-4.png" alt="选择存储设备" width="80%" />

有关网络安装配置的更多信息，请参阅[HTTP boot](./raspberry-pi.mdx#http启动)。

## 设置你的树莓派

安装操作系统映像后，将存储设备插入（或连接到）树莓派。

首先，拔掉树莓派的电源，以确保在连接外围设备时树莓派处于断电状态。如果您将操作系统安装在microSD卡上，您现在可以将其插入树莓派的卡插槽。如果您在其他存储设备上安装了操作系统，您现在可以将其连接到树莓派。

<img src="/img/computers/getting-started/peripherals/sd-card.png" alt="将microSD卡插入树莓派" width="80%" />

然后，插入其他外围设备，例如鼠标、键盘和显示器。

<img src="/img/computers/getting-started/peripherals/cable-all.png" alt="将电源连接到树莓派" width="80%" />

最后，将电源连接到树莓派。当树莓派开机时，您应该会看到状态LED亮起。如果您的树莓派连接到显示器，您应该会在几分钟内看到启动屏幕。

## 首次启动配置
若您已在Imager安装过程中自定义了树莓派操作系统，恭喜您，设备已准备就绪。继续执行[下一步](#下一步)，了解如何充分利用树莓派。

如果您的树莓派在5分钟内未启动，请检查状态LED。如果闪烁，请参阅[LED警告闪烁代码](./configuration.mdx#led警告闪烁代码)以获取更多信息。如果您的树莓派不能启动，请尝试以下缓解步骤：

- 如果您使用SD卡以外的启动设备，请尝试从SD卡启动
- [重新刷新映像到您的SD卡](#安装操作系统)；确保在Imager中完成整个验证步骤
- [更新树莓派上的bootloader](./raspberry-pi.mdx#更新-bootloader)，然后[重新刷新映像到您的SD卡](#安装操作系统)

如果您选择跳过Imager中的自定义选项，您的树莓派将在首次启动时将运行配置向导。您需要显示器和键盘来查看向导；鼠标是可选的。
