# 触摸显示屏

## 树莓派触摸显示屏

[树莓派触摸显示屏](https://www.raspberrypi.com/products/raspberry-pi-touch-display/) 是一款 7 英寸触摸屏 LCD，适用于平板电脑、娱乐系统和信息仪表盘等交互式项目。

<img src="/img/accessories/display/display.png" alt="树莓派7英寸触摸显示屏" width="70%" />
*树莓派7英寸触摸显示屏*

该显示屏采用金属框架背板，并预装了树莓派显示转换板和金属支座。可[在此处](https://pip.raspberrypi.com/documents/RP-008246-DS)下载触摸显示屏的机械图纸。

## 兼容性

触摸显示屏兼容 Raspberry Pi B+ 及更高版本。

不兼容 Raspberry Pi Zero 系列或 Raspberry Pi Keyboard 电脑，因为它们没有 DSI 连接器。

Raspberry Pi 5 使用不同尺寸的 DSI 连接器，但您可以使用 [22 路转 15 路扁平柔性线缆](https://www.raspberrypi.com/products/display-cable/)（需另购）将其连接到触摸显示屏。

Raspberry Pi 5 和 Raspberry Pi Compute Module 原生支持连接最多两个触摸显示屏。

您可以将任何 Raspberry Pi B+ 及更高版本安装到触摸显示屏背面。但安装 Raspberry Pi B 或更早版本时，需要额外的安装硬件来将支座安装到显示 PCB 上。

## 自动检测

Raspberry Pi B+ 及更高版本设备会自动检测触摸显示屏。

Raspberry Pi A 和 B 可以自动检测显示屏，但该功能默认关闭，因此您必须手动启用它。

Raspberry Pi Compute Module 不会自动检测 DSI 显示屏，因为它们专为工业应用设计，集成商可以将任何和所有 GPIO 及接口用于自己的目的。

## 规格说明

本节介绍触摸显示屏的特性和能力。

### 功能特性

触摸显示屏具有以下功能：

- **800 × 480px RGB LCD** 显示屏
- **24 位 RGB 显示**：可呈现超过 1600 万种色彩
- **多点触控**：最多支持十个同时触控点
- **屏幕键盘**：支持使用屏幕键盘替代物理键盘
- **集成供电**：由主机树莓派设备直接供电，无需独立电源
- **背光寿命**：20,000 小时

### 视角性能

- **工业品质**：水平视角 140 度，垂直视角 120 度
- **PWM 背光控制和电源控制**：通过 I2C 接口
- **对比度**：500
- **平均亮度**：250 cd/m²
- **可视区域**：154.08 × 85.92 毫米
- **视角（度）**：
  - 顶部 - 50
  - 底部 - 70
  - 左 - 70
  - 右 - 70

### 电源

- **功耗**：最大亮度时，典型值为 5V 200mA。

### 尺寸与环境参数

- **工作温度**：-20 至 +70 摄氏度
- **存储温度**：-30 至 +80 摄氏度
- **外部尺寸**：192.96 × 110.76 毫米

### 硬件

触摸显示屏包装盒内含以下物品：

- 一块触摸显示屏
- 四个支座和螺丝（预装但可拆卸）
- 一条 15 路转 15 路扁平柔性线缆
- 四根跳线

## 将树莓派安装到触摸显示屏

您可以将任何 Raspberry Pi B+ 及更高版本安装到触摸显示屏背面。

如果您有单独的机箱，也可以将触摸显示屏安装到机箱中。连接方式保持不变，但您可能需要更长的线缆。

<img src="/img/accessories/display/GPIO_power-500x333.jpg" alt="连接到触摸显示屏的树莓派" width="100%" />
*连接到触摸显示屏的树莓派*

要将设备安装到显示屏上：

1. 向外滑动触摸显示屏上 FFC 连接器两侧的固定夹。
2. 将扁平柔性线缆 (FFC) 的一端连接到触摸显示电路板上的 `RPI-DISPLAY` 端口，然后将固定夹推回原位以固定线缆。

    银色或金色触点应远离显示屏。

3. 向外滑动设备上 FFC 连接器两侧的固定夹。
4. 将 FFC 的另一端连接到树莓派上的 `DISPLAY` 端口，然后将固定夹推回原位以固定线缆。

    触点应朝内，朝向树莓派。

5. 将树莓派的四个角支座与触摸显示屏背面的四个安装点对齐。
6. 将 M2.5 螺丝（随附）插入四个角支座并拧紧，直到树莓派牢固固定。

如果 FFC 没有完全插入或位置不正确，您会遇到显示问题。开机时如果显示屏空白或只显示单色，请仔细检查此连接。

## 连接到 Raspberry Pi B+ 及更高版本

我们建议使用树莓派的 GPIO 为触摸显示屏供电。

或者，您也可以使用单独的微型 USB 电源直接为显示屏供电。

::::warning[警告]
开始之前，请确保树莓派已关闭电源并断开电源连接。
::::

### 从树莓派供电

要使用树莓派为触摸显示屏供电：

1. 在[树莓派 GPIO](../computers/raspberry-pi.mdx#gpio) 的 5V 和 `GND` 引脚与显示屏的 5V 和 `GND` 引脚之间连接跳线。

    <img src="/img/accessories/display/display_plugs.png" alt="显示屏引脚示意图" width="40%" />
    *显示屏 5V 和 GND 引脚的位置*

2. 将黑色跳线的一端连接到树莓派的第六针（`GND`），红色跳线的一端连接到第四针（5V）。

    如果第六针不可用，请使用任何其他开放的 `GND` 针连接黑线。

    如果第四针不可用，请使用任何其他 5V 针连接红线，例如第二针。

    <img src="/img/accessories/display/pi_plugs.png" alt="树莓派针座示意图" width="40%" />
    *树莓派针座的位置*

3. 将黑线的另一端连接到显示屏上的 `GND` 引脚，红线的另一端连接到显示屏上的 5V 引脚。
4. 将树莓派连接到电源，然后开机。触摸显示屏即会打开。

### 从微型 USB 电源供电

如果不想使用树莓派为触摸显示屏供电，可以使用微型 USB 电源。

我们建议使用[树莓派 12.5W 电源](https://www.raspberrypi.com/products/micro-usb-power-supply/)，以确保显示屏按预期运行。

如果选择使用微型 USB 供电，请不要将树莓派上的 GPIO 引脚连接到显示屏。两块板之间的唯一连接应该只有 FFC。

::::warning[警告]
使用微型 USB 电缆为显示屏供电时，请将其安装在机箱内，以防止在使用过程中接触到显示屏的印刷电路板。
::::

## 连接到 Raspberry Pi 1 及更早版本

这些说明仅适用于原版树莓派，即 Model A 和 B 板。

要识别原版树莓派，请检查 GPIO 排针：只有原版型号有 26 针 GPIO 排针；后续型号有 40 针。

树莓派 1 Model A 和 B 板上的 DSI 连接器缺少与触摸屏控制器和 DSI 控制器通信所需的 I2C 连接。解决方法是使用显示套件提供的附加跳线。

这些板默认禁用 DSI 显示自动检测功能。

要连接到 Raspberry Pi Model B 及更早版本：

1. 将以下行添加到 `/boot/firmware/config.txt` 文件中以启用树莓派的自动检测功能：

    ```ini
    ignore_lcd=0
    ```

2. 关闭设备电源，然后拔掉电源。
3. 将 GPIO 排针上的 SCL/SDA 连接到显示板上标有 `SCL/SDA` 的水平引脚。
4. 使用 GPIO 引脚和随附的跳线，从树莓派为触摸显示屏供电。
5. 将树莓派连接到电源，然后开机。触摸显示屏即会打开。
6. 通过显示板上的 `PWR IN` 微型 USB 接口为设备供电。

    ::::warning[警告]
    请勿通过树莓派的微型 USB 端口供电。这将超过输入保险丝的最大额定电流，因为显示屏功耗约为 400mA。
    ::::

## 连接到 Raspberry Pi Compute Module {#connect-to-cm}

触摸显示屏的自动检测功能在 Compute Module 上已禁用。这意味着设置显示屏所需的设备树片段必须通过以下方式之一加载：

- 在 config.txt 中添加 dtoverlay 条目。
- 自定义基础 DT 文件。
- HAT EEPROM（如果存在）。

开始之前，请将您的 Raspberry Pi Compute Module 的系统和固件更新到最新版本。

::::warning[警告]
开始之前，请确保 Raspberry Pi Compute Module 已关闭电源并断开电源连接。
::::

### 将显示屏连接到 DISP1/DSI1

要将显示屏连接到 `DISP1/DSI1`：

1. 通过 22 路转 15 路显示适配器将显示屏连接到 Compute Module IO 板上的 `DISP1/DSI1` 端口。
2. 完成适当的跳线连接：
   - 对于 **CM1**、**CM3**、**CM3+** 和 **CM4S**，使用跳线连接以下 GPIO 引脚：
     - `0` 到 `CD1_SDA`
     - `1` 到 `CD1_SCL`
   - 对于 **CM5**，在 Compute Module 5 IO 板上，在 J6 上添加适当的跳线，如丝印所示。
3. 将 Raspberry Pi Compute Module 连接到电源，然后开机。触摸显示屏即会打开。
4. 将 `dtoverlay=vc4-kms-dsi-7inch` 添加到 `/boot/firmware/config.txt`。
5. 使用 `sudo reboot` 重启 Compute Module。您的设备将检测并在显示屏上显示输出。

### 将显示屏连接到 DISP0/DSI0

要将显示屏连接到 CM1、CM3 和 CM4 IO 板上的 `DISP0/DSI0`：

1. 通过 22 路转 15 路显示适配器将显示屏连接到 Compute Module IO 板上的 `DISP0/DSI0` 端口。
2. 完成适当的跳线连接：
   - 对于 **CM1**、**CM3**、**CM3+** 和 **CM4S**，使用跳线连接以下 GPIO 引脚：
     - `28` 到 `CD0_SDA`
     - `29` 到 `CD0_SCL`
   - 对于 **CM4**，在 Compute Module 4 IO 板上，在 J6 上添加适当的跳线，如丝印所示。
3. 重新连接 Compute Module 电源。
4. 将 `dtoverlay=vc4-kms-dsi-7inch,dsi0` 添加到 `/boot/firmware/config.txt`。
5. 使用 `sudo reboot` 重启 Compute Module。您的设备将检测并在显示屏上显示输出。

## 禁用触摸屏

要禁用触摸功能但仍使用显示屏，请在 `/boot/firmware/config.txt` 中添加以下行：

```ini
disable_touchscreen=1
```

## 禁用显示屏

要在连接时完全忽略显示屏，请在 `/boot/firmware/config.txt` 中添加以下行：

```ini
ignore_lcd=1
```

## 使用屏幕键盘

Raspberry Pi OS _Bookworm_ 及更高版本默认包含 Squeekboard 屏幕键盘。当连接触摸显示屏时，屏幕键盘会在可输入文本时自动显示，在无法输入文本时自动隐藏。

对于不支持文本输入检测的应用程序，可使用任务栏右端的键盘图标手动显示或隐藏键盘。

您还可以在控制中心的 **显示** 选项卡或 `raspi-config` 的 `显示` 部分中永久显示或隐藏屏幕键盘。

::::tip
在 _Bookworm_ 之前的 Raspberry Pi OS 版本中，请使用 `matchbox-keyboard` 代替。如果使用 Wayfire 桌面合成器，请使用 `wvkbd` 代替。
::::

## 更改屏幕方向

如果您想旋转显示屏或将其安装在特定位置，请从 **首选项** 菜单中选择 **屏幕配置**。右键单击布局编辑器中的触摸显示矩形（可能是 DSI-1），选择 **方向**，然后选择最适合您需要的选项。

<img src="/img/accessories/display/display-rotation.png" alt="屏幕配置中方向选项的屏幕截图" width="80%" />

### 在没有桌面的情况下旋转屏幕

要在没有桌面环境的设备上设置屏幕方向，请编辑 `/boot/firmware/cmdline.txt` 配置文件，将方向传递给系统。在 `cmdline.txt` 中添加以下行：

```ini
video=DSI-1:800x480@60,rotate=<rotation-value>
```

将 `<rotation-value>` 占位符替换为以下值之一，这些值与显示屏上相对于默认值的旋转度相对应：
- 0
- 90
- 180
- 270

例如，旋转值为 `90` 时，显示屏会向右旋转 90 度。`180` 则将显示屏旋转 180 度，即上下颠倒。

::::note
使用 `cmdline.txt` 无法将 DSI 显示屏与 HDMI 显示屏分开旋转。同时使用 DSI 和 HDMI 时，它们共享相同的旋转值。
::::

## 旋转触摸输入

::::warning[警告]
通过设备树旋转触摸输入可能会与输入库发生冲突。请尽可能在输入库或桌面中配置触摸事件旋转。
::::

触摸输入的旋转与显示屏本身的方向无关。要更改这一点，需要在 `/boot/firmware/config.txt` 中手动添加 `dtoverlay` 指令。在 `config.txt` 中最后添加以下行：

```ini
dtoverlay=vc4-kms-dsi-7inch,invx,invy
```

然后，从 `config.txt` 中删除以下行（如果存在），禁用自动显示检测功能：

```ini
display_auto_detect=1
```

## 触摸显示屏设备树选项参考

`vc4-kms-dsi-7inch` 叠加支持以下选项：

| DT参数 | 操作 |
| -- | -- |
| `sizex` | 设置 X 分辨率（默认为 800） |
| `sizey` | 设置 Y 分辨率（默认为 480） |
| `invx` | 反转 X 坐标 |
| `invy` | 反转 Y 坐标 |
| `swapxy` | 交换 X 和 Y 坐标 |
| `disable_touch` | 禁用触摸覆盖层 |

要指定这些选项，请将它们添加到 `/boot/firmware/config.txt` 中的 `dtoverlay` 行，中间用逗号隔开。布尔值出现时默认为 true，但可以使用后缀 `=0` 将其设置为 false。整数值需要一个值，例如 `sizey=240`。例如，要将 X 分辨率设置为 400 像素并反转 X 和 Y 坐标，请使用下面一行：

```ini
dtoverlay=vc4-kms-dsi-7inch,sizex=400,invx,invy
```
