# 投稿及问题反馈

欢迎向代码库投稿！以下是一些应遵循的准则。

## 建议

请就代码库的附加组件或增强功能提出建议，并在 [问题](https://github.com/gpiozero/gpiozero/issues/new) 中清楚地解释您的理由。

## 错误

请提交错误报告，打开一个 [问题](https://github.com/gpiozero/gpiozero/issues/new)，使用代码示例清楚地解释问题。

## 文档

文档源存在 [docs](https://github.com/gpiozero/gpiozero/tree/master/docs) 文件夹中。我们欢迎您为文档撰稿，但文档应易于阅读和理解。

## 提交信息和拉取请求

提交信息应简明扼要，采用补丁描述的形式，即指示式而非过去式（"Add LED example" 而非 "Added LED example"）。

关闭（或打算关闭）某个问题的提交应包含 "fix #123 "或 "close #123"，其中 `#123` 是问题编号，同时还应包含简短的描述，例如："Add LED example, close #123"（添加 LED 示例，关闭 #123），拉取请求应与相应的问题标题相匹配或接近。

提交内容的版权归作者所有（我们不需要进行版权转让），我们假定作者乐意在项目 [授权协议](./license.mdx) 下发布他们的代码。请随时在您的拉取请求中将您的名字添加到项目顶层 `README.rst` 中的贡献者列表中！不用担心在您所接触的任何文件的版权标题中添加您的名字，每次发布前，git 元数据都会自动更新这些内容。

## 向后兼容

自本库发布 v1.0 以来，我们一直致力于保持向后兼容性。不接受破坏向后兼容性的修改。

## Python 2/3

从 3.2 版本开始，该库与 Python 2.7 和 Python 3 100% 兼容。由于 Python 2 已经过了 [生命周期的最后期限](http://legacy.python.org/dev/peps/pep-0373/)，1.6.2 版本 (2021-03-18) 是最后一个支持 Python 2 的版本。
